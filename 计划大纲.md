# **高校教师信息智能化管理系统设计与实现**

1. ## **确定需求**

   网上自动爬取数据，按照大学、学院、专业自动爬取，并生成数据库以及前台界面，供用户终端查询。显示教师姓名、性别、研究方向、邮箱等信息。

   安卓APP、电脑端、微信小程序皆可使用

2. ## 系统架构与技术栈

   **前端：**

   - PC Web端： Vue.js / React + Element UI / Ant Design（现代化， 开发效率高）。
   - Android App： Kotlin（原生）或 React Native / Flutter（跨平台， 节省成本）。
   - 微信小程序： 微信小程序原生框架。

   **后端**： 

   - 语言： Python（爬虫生态强大，Web框架成熟）或 Java（企业级应用稳定）。
   - 框架： Django（Python， 自带ORM和Admin， 开发快）或 Spring Boot（Java， 生态完善）。
   - 爬虫框架： Scrapy, Requests, BeautifulSoup, Selenium（对付动态加载的页面）。
   - API： RESTful API， 供所有前端调用。

   **数据库**：MySQL（存储教师信息等结构化数据）

   | **表名**    | **字段**                                                     | **说明**         |
   | ----------- | ------------------------------------------------------------ | ---------------- |
   | university  | id, name, province, website                                  | 高校信息         |
   | college     | id, university_id, name                                      | 学院信息         |
   | teacher     | id, name, gender, title, email, phone, research_direction, university_id, college_id | 教师信息         |
   | user        | id, username, password, role                                 | 管理员与普通用户 |
   | test_result | id, module, success_rate, error_count, date                  | 测试结果记录     |

   **部署与运维：**

   - 服务器： 云服务器（如阿里云ECS， 腾讯云CVM）。
   - 部署： Docker容器化部署。
   - 任务调度： Celery（用于定时爬取任务）。

   **ai自动测试：**Python+Selenium（自动生成并运行测试脚本）

   ​

3. ## 步骤拆分

   1. 功能设计分析，项目框架初始化，环境搭建
   2. 前后端开发
      1. 数据爬取＋处理：
         - 针对目标编写爬虫脚本
         - 处理反爬虫
         - 格式化收集到的内容并存入数据库
      2. API服务模块
      3. pc web端前端开发
      4. android+小程序前端开发
   3. 生成测试脚本
   4. debug
   5. 撰写报告
